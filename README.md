# SA-OFFICE

다우오피스 스타일의 사내 업무 시스템입니다. Next.js + Prisma + PostgreSQL로 만들었습니다.

## 실행 방법 (새 컴퓨터에서 시작할 때는 이 폴더를 통째로 새로 풀어주세요 - 기존 폴더에 덮어쓰지 마세요)

```bash
npm install
cp .env.example .env       # DATABASE_URL을 본인 Neon/Postgres 주소로 채워주세요
npx prisma db push
npm run db:seed
npm run dev
```

브라우저에서 http://localhost:3000 접속 → 로그인 화면으로 이동합니다.

## 3단계 권한 체계

- **사용자** — 기본값. 새 직원 계정을 등록하면 자동으로 이 등급이 부여됩니다.
- **관리자 (MANAGER)** — 권한 부여 가능(관리 > 권한), 공지사항 작성 가능. 단 admin 등급을 새로 부여할 수는 없고, 이미 admin인 사용자의 권한도 바꿀 수 없습니다.
- **admin** — 전체 기능 (사용자 등록, 프로젝트 관리, 권한 부여, 공지사항 작성 등 전부).

권한 변경은 `/admin/permissions`에서 할 수 있고, 본인 스스로의 권한은 바꿀 수 없습니다.

## 전체 구조

- `/login` — 로그인 (좌측 이미지, 우측 아이디/비밀번호)
- `/` — 메인 캘린더
- `/notices` — 공지사항 (누구나 조회 가능, admin/관리자만 작성 가능)
- `/board` — 게시판 (준비중)
- `/approval` — 전자결재 (기안함/결재함/진행함/완료함/반려함)
- `/approval/new` — 새 기안 작성 (기안자는 항상 로그인한 나 자신)
- `/admin/users` — (admin 전용) 신규 직원 계정 등록 → 등록 즉시 "사용자" 권한 자동 부여
- `/admin/projects` — (admin 전용, 준비중)
- `/admin/permissions` — (admin, 관리자 모두 접근) 사용자별 권한 변경

로그인하지 않은 상태로 어떤 페이지에 접속해도 자동으로 `/login`으로 이동합니다 (middleware에서 처리).
권한이 부족한 사용자가 `/admin/*` 에 접속하면 홈으로 리다이렉트됩니다.

## 인증 방식 (중요 - 학습용 MVP 한계)

- 비밀번호는 bcrypt로 해시해서 저장 (이 부분은 실제 서비스 수준)
- 로그인 세션은 쿠키에 "누구인지" 정보를 서명 없이 base64로만 담아서 저장합니다. 실제 서비스로 전환하려면 NextAuth.js 또는 서명된 JWT로 교체하는 걸 추천합니다.
